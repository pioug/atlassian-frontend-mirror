## Usage

Use tags to classify, organize, and categorize information. Tags work best for lower priority
metadata such as categories, attributes, or topics that support scanning and filtering. They can be
rendered as non-interactive, or as interactive elements to support linking and removal with a close
button. If you need to represent a status or state, for example "In progress", use a
[lozenge](/components/lozenge/lozenge) instead.

Tags are designed to be displayed within a [tag group](/components/tag-group/tag-group). Use a tag
group to control the layout and alignment for a collection of tags.

1. **Label:** Text that describes the classification or category.
2. **Leading element (optional):** Additional meaning can be added with an icon or another element.
   If using an icon, make sure it has a clear and well-established meaning that relates to the tag
   label.
3. **Close button (optional):** Enable the `isRemovable` property to allow customers to remove a tag
   by clicking the trailing close button.

## Best practices

### Only use tags for object-related content

Tags are reserved for object related content. For people, team, project or space avatars, use
[avatar tag](/components/tag/avatar-tag).

	}}
>
	Use tags for object-oriented elements.
</DoDont>

### Don't use tags to communicate status or state

Tags should not be used to classify, organize, and highlight information. Tags provide less visual
prominence than lozenges to create visual hierarchy. To communicate status or state use
[lozenge](/components/lozenge/lozenge) instead.

<DoDont
	type="do"
	image={{
		url: tagDo2Light,
		urlDarkMode: tagDo2Dark,
		alt: 'Tags showing categories: Design, Lead, Marketing, Management, Backend',
	}}
>
	Use tags to classify, organize, and highlight information.
</DoDont>
<DoDont
	type="dont"
	image={{
		url: tagDont2Light,
		urlDarkMode: tagDont2Dark,
		alt: 'Tags showing status labels: Off track, In progress, Completed, In review, and strikethrough with color',
	}}
>
	Don't use tags to communicate the status of an item.
</DoDont>

### Use color with intention

Use tag color to thematically organize related content. If the same color represents multiple
categories, its meaning becomes ambiguous. Apply color intentionally and avoid overlap to maintain
clarity.

### Don’t use tags within user-generated text

Tags are reserved for labelling and should not be used in place of content in user-generated
long-form text or prompts, such as in editor.

## Content guidelines

- When writing tags, avoid line wrapping.
- Tag has a maxWidth of 200 pixels (or a custom maxWidth value) causing labels to truncate. If the
  tag isn't user generated, keep labels short to avoid truncation.
