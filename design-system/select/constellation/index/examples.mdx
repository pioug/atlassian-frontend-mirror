## Named exports

To consolidate patterns and improve developer experience,
select provides some pre-configured components as named exports.

### Popup select

Use `PopupSelect` in scenarios where the trigger needs a custom appearance,
or must occupy very limited horizontal space.

<Example Component={SelectPopup} packageName="@atlaskit/select" />

### Radio select

Use `RadioSelect` to display the list of options as radio elements.

<Example Component={SelectRadio} packageName="@atlaskit/select" />

### Checkbox select

Use `CheckboxSelect` to display the list of options as checkbox elements.
Use this when it is imperative to show multiple selected options, but there may not be enough real-estate in the control.

<Example Component={SelectCheckbox} packageName="@atlaskit/select" />

### Country select

Use `CountrySelect` to add a country picker to the dropdown list of options.

<Example Component={SelectCountry} packageName="@atlaskit/select" />

### Async select

Use `AsyncSelect` to load asynchronous data from a remote source either during load or every time the input changes.

<Example Component={SelectAsync} packageName="@atlaskit/select" />

### Creatable select

Use `CreatableSelect` to allow the user to insert an item into the list for selection.

<Example Component={SelectCreatable} packageName="@atlaskit/select" />

### Async creatable

Use `AsyncCreatableSelect` to allow users to create new options asynchronously.
For example, execute async calls before creating the option.

<Example Component={SelectAsyncCreatable} packageName="@atlaskit/select" />
