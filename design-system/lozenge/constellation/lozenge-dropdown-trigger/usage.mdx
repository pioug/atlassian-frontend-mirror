## Usage

Lozenge dropdown triggers indicate the status of items while also triggering a menu to enable status
switching. They differ from lozenges in that they contain interactivity and can be selected.

## Anatomy

<Image
	src={lozengeDropDownAnatomyLight}
	srcDark={lozengeDropDownAnatomyDark}
	alt="Lozenge dropdown anatomy diagram."
/>

1. **Icon (optional):** Use an icon to add additional affordance where the icon has a clear and
   well-established meaning. For example, arrow up right for “On track”.
2. **Label**: Text describing the status. For example, “On track”.
3. **Badge (optional)**: Use the trailingMetric prop to add a score to the status.
4. **Dropdown icon**: Indicates that this lozenge has interactivity and can be selected to open a
   menu.

## Best practices

### Use lozenge dropdown trigger for status switching

Lozenge dropdown trigger should only open a dropdown menu or a popup to allow for quick status
switching.

elements, such as when it appears alongside other buttons.

## Accessibility

- Don't use color alone to signify an important state or status. Instead, use clear labels and, if
  relevant, a supporting icon. For example: for a critical status, use descriptors like ’Off track’.
- Don't use long labels in lozenges. They have a maxWidth of 200 pixels (or a custom maxWidth value)
  causing labels to truncate. Lozenges can’t be focused, so truncated text will not be visible.
