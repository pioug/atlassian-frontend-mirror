## Usage

Lozenges indicate the status or state of objects, such as the progress of a work item. They help
people quickly understand an object’s state, so it’s important they are used accurately and
consistently.

## Anatomy

<Image
	src={lozengeAnatomyLight}
	srcDark={lozengeAnatomyDark}
	alt="Lozenge anatomy diagram showing: 1. Label (text that describes the status. For example, “On track”), 2. Icon (optional to add additional affordance), and 3. Badge (optional, to add a score to the status)"

1. **Label**: Text describing the status. For example, “On track”.
2. **Icon (optional)**: Use an icon from the iconography library to add additional affordance where
   the icon has a clear and well-established meaning. For example, arrow up right for “On track”.
3. **Badge (optional)**: The `trailingMetric` prop can be utilised on semantic colored lozenges to
   add a score to the status.

## Best practices

### Only use lozenge to communicate status or state

To classify, organize, or categorize information, use a [tag](/components/tag) instead. Tags have
lower visual prominence than lozenges to create a clear visual hierarchy that prioritizes important
information that needs customer attention.

### Use accent and semantic colors correctly

When choosing a lozenge color, there’s two sets of color choices available:

- **Semantic colors (for example, "success")**: use for status or state where color usage reflects
  semantic intent.
- **Accent colors (for example, "accent-red")**: use when color isn’t used for semantic intent. In
  most cases, accent colors are used for user-generated content, where customers can choose a color
  and the text label.

lozenge. Use it to add a score to a status, for example for a goal.

### Use lozenge dropdown trigger when interactivity is required

Lozenges are non-interactive. If you require a lozenge that opens a dropdown menu, for example an
inline status switcher, use a
[lozenge dropdown trigger](/components/lozenge/lozenge-dropdown-trigger).

## Content guidelines

### Use sentence case capitalization

Only capitalize the first letter of the label and any proper nouns. View our
[capitalization guidance](/foundations/content/language-and-grammar#capitalization) for more
information.

## Accessibility

- Don't use color alone to signify an important state or status. Instead, use clear labels and, if
  relevant, a supporting icon. For example: for a critical status, use descriptors like “Off track”.
- Don't use long labels in lozenge. They have a max width of 200 pixels (or a
  custom `maxWidth` value) causing labels to truncate. Lozenges can’t be focused, so truncated text
  will not be visible.
