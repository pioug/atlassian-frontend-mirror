## Component examples

### Button

This primary button uses a background token, text token, and other tokens to create a brand appearance.

Buttons and other components can have multiple possible [interaction states](/foundations/color/#interaction-states), which require specific tokens to be applied for each state.

<TokensButtonExample/>

### Modal dialog

Modal dialogs use a surface token and shadow token to illustrate [elevation](/foundations/elevation). The surface token ensures no content shows underneath the modal.

<TokensModalExample/>

### Tag

In the tag example, we can see the use of accent colors. Accent colors are available for text, icons, borders and backgrounds. Apply accent tokens where [colors](/foundations/color/#emphasis-level) do not have a specific meaning tied to them.

<TokensTagExample/>

### Table

An overflow shadow can be applied on horizontal scrollable tables to indicate there is hidden content that can be scrolled back into view.

<TokensTableExample/>
